# Security Policy

## Supported Versions

We currently support the latest major version of the project. If you are using an older version, we recommend upgrading to receive the latest security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.x.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please do not disclose it publicly. Instead, report it privately to the maintainers so we can address it safely.

You can report vulnerabilities by:
- Sending an email to the project maintainers.
- Creating a private vulnerability report in the GitHub repository via the **Security** tab > **Advisories** > **Report a vulnerability** (if enabled).

Please include the following details in your report:
- A clear description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact of the vulnerability.

We take all security reports seriously and will acknowledge your report within 48 hours. We will keep you updated on the progress of the fix and aim to resolve confirmed vulnerabilities promptly.
