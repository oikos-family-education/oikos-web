# 8. Development

How to run, build, test, and migrate Oikos locally.

## Prerequisites

- **Node.js 20+** and **npm 10+**
- **Python 3.12** with `venv`
- **Docker + Docker Compose** (recommended for the full stack)
- **PostgreSQL 16** and **Redis 7** if you run without Docker

## Clone & install

```bash
git clone <repo>
cd oikos-web
npm install            # installs root + workspaces
```

## Environment variables

Each app reads its own `.env`. Examples:

`apps/api/.env`
```
DATABASE_URL=postgresql+asyncpg://oikos:oikos@db:5432/oikos
REDIS_URL=redis://redis:6379/0
JWT_SECRET_KEY=change-me
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=30
CORS_ORIGINS=["http://localhost:3000"]
```

`apps/web/.env.local`
```
NEXT_PUBLIC_API_BASE_URL=/api
```

In host-mode (no Docker) swap `db` and `redis` for `localhost`.

## Running

### Full stack via Docker (recommended)

```bash
docker compose up -d
# Web  → http://localhost:3000
# API  → http://localhost:8000  (/docs for Swagger)
```

Alembic migrations run automatically via the `migrate` service.

### Backing services only, apps on host

```bash
docker compose up -d db redis

# Backend
cd apps/api
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload        # :8000

# Frontend (new terminal)
cd apps/web
npm run dev                           # :3000
```

### Everything via Turborepo

```bash
npm run dev         # boots both apps via turbo
```

## Scripts reference

### Root (`package.json`)

| Command | What it does |
|---|---|
| `npm run dev` | `turbo dev` across workspaces |
| `npm run build` | `turbo build` |
| `npm run lint` | `turbo lint` |
| `npm run test` | `turbo test` |
| `npm run type-check` | `turbo type-check` |

### Frontend — `apps/web`

| Command | What it does |
|---|---|
| `npm run dev` | Next.js dev server (port 3000) |
| `npm run build` | Production build |
| `npm run start` | Start a built server |
| `npm run lint` | ESLint |
| `npm run test` | Vitest |

### Backend — `apps/api`

| Command | What it does |
|---|---|
| `uvicorn app.main:app --reload` | Dev server (port 8000, auto-reload) |
| `pytest` | Run all tests against a real DB |
| `pytest tests/test_auth.py` | Single test file |
| `alembic upgrade head` | Apply pending migrations |
| `alembic revision --autogenerate -m "msg"` | Create a migration from model diff |

## Creating a database migration

```bash
cd apps/api
# 1. Change the SQLAlchemy model
# 2. Autogenerate a migration
alembic revision --autogenerate -m "add foo table"
# 3. Review the generated file in alembic/versions/
# 4. Apply
alembic upgrade head
```

In Docker: `docker compose exec api alembic upgrade head`. See [05-database.md](05-database.md).

## Troubleshooting

- **401 loop on the frontend.** Clear cookies; the refresh token may be expired or mis-signed (check `JWT_SECRET_KEY` parity between runs).
- **Migration service exits non-zero on boot.** `docker compose logs migrate` — usually a DB not yet ready or a conflicting autogenerated revision.
- **`/api/*` returns HTML (Next 404).** The proxy target is wrong; confirm `next.config.js` rewrites and that the API container is up.
- **Rate-limited on login during dev.** Flush Redis: `docker compose exec redis redis-cli FLUSHDB`.
